# Synthetic Benchmark Data

This directory contains synthetically generated performance benchmark data for development and testing of the dashboard.

## Generation Approach

The synthetic data is generated by `src/synthetic_data.py` and mirrors the structure of real benchmark results from the OpenSearch `zathras-results` index.

### Data Characteristics

- **Total Documents**: 150
- **Scenarios**: 30 unique configurations
- **Iterations**: 5 test runs per scenario
- **Time Span**: 90 days of historical data

### Dimensions Covered

#### Operating Systems
- RHEL 9.3
- RHEL 9.4
- RHEL 9.5
- RHEL 9.6

#### Cloud Providers & Instance Types
- **AWS**: m5.24xlarge, m5.12xlarge, m5.8xlarge, c5.24xlarge, c5.12xlarge
- **Azure**: Standard_D96s_v3, Standard_D48s_v3, Standard_F96s_v2
- **GCP**: n2-highmem-96, n2-highmem-64, c2-standard-60

#### Benchmark Types
- `coremark` - CoreMark CPU benchmark
- `coremark_pro` - CoreMark-PRO suite
- `passmark` - PassMark Performance Test
- `streams` - STREAM memory bandwidth
- `auto_hpl` - High-Performance Linpack
- `pyperf` - Python performance benchmarks
- `phoronix` - Phoronix Test Suite
- `uperf` - Network performance
- `pig` - Apache Pig benchmarks

### Performance Patterns

The synthetic data includes three types of performance patterns:

1. **Stable** (70%): Performance varies within ±5%
   - Represents normal test variance
   - Most common in production environments

2. **Improvement** (15%): Performance increases 15-30%
   - Simulates OS or hardware improvements
   - Optimization gains

3. **Regression** (15%): Performance decreases 20-40%
   - Simulates performance regressions
   - Some marked as FAIL status

### Metric Value Ranges

Baseline values are derived from actual OpenSearch data samples:

| Benchmark | Primary Metric | Typical Range | Unit |
|-----------|---------------|---------------|------|
| CoreMark | multicore_score | 250,000 - 650,000 | BOPs |
| CoreMark-PRO | SUMM_CPU_mean | 27,500 - 71,500 | BOPs |
| PassMark | CPU_INTEGER_MATH | 135,000 - 351,000 | score |
| STREAM | copy__mb_per_sec | 90,000 - 234,000 | MB/s |
| HPL | gflops | 1,500 - 3,250 | GFLOPS |
| Pyperf | mean | 0.25 - 0.65 | seconds |

### Statistical Properties

Each metric includes:
- `_mean`: Average value
- `_min`: Minimum observed value (≈ 98% of mean)
- `_max`: Maximum observed value (≈ 102% of mean)
- `_stddev`: Standard deviation (≈ 1% of mean)

## Regenerating Data

To regenerate the synthetic data with different parameters:

```bash
python src/synthetic_data.py
```

Or programmatically:

```python
from src.synthetic_data import SyntheticDataGenerator

generator = SyntheticDataGenerator(seed=42)
documents = generator.generate_dataset(
    num_scenarios=50,  # More configurations
    iterations_per_scenario=10  # More runs per config
)
generator.save_to_file(documents, "data/synthetic/custom_data.json")
```

## Use Cases

1. **Development**: Test dashboard features without OpenSearch connection
2. **Testing**: Validate data processing and visualization logic
3. **Demonstrations**: Show dashboard capabilities with sample data
4. **Performance**: Benchmark dashboard responsiveness with known data volume

## Validation

The synthetic data:
- ✓ Matches real OpenSearch schema structure
- ✓ Uses realistic metric value ranges
- ✓ Includes multiple OS versions and hardware configs
- ✓ Contains both PASS and FAIL test results
- ✓ Simulates temporal distribution (90-day window)
- ✓ Includes run-to-run variation

## Limitations

- CPU flags and detailed hardware specs are simplified
- Test-specific configuration parameters are minimal
- No actual test failure logs or error details
- Fixed number of test runs (real data varies)
- Simplified NUMA topology
